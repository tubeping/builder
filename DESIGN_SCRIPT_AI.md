## 1. 타겟 페르소나 `[미확정]`

### 결정해야 할 변수
- **구독자 규모**: 1천~1만 / 1만~10만 / 10만+ / 전 구간
- **공구 경험**: 첫 공구 / 여러 번 / 둘 다
- **주 플랫폼**: 유튜브 / 인스타 / 틱톡 / 복수
- **주 카테고리**: 뷰티 / 건강 / 리빙 / 식품 / 육아 / 패션 / 전방위

### 기본값 (답 없을 시 가정)
```
구독자:   1만~10만 (공구를 정기적으로 하는 허리)
경험:     여러 번 (자기 패턴 있음, 효율화 원함)
플랫폼:   유튜브 롱폼 + 쇼츠 동시 운영
카테고리: 뷰티 · 건강 · 리빙 중심
```

## 2. "적합한 스크립트"의 기준 `[미확정]`

### 두 갈래

**A타입 — 편집하는 초안**
- AI 70% · 크리에이터 30%
- 구조·공구 논리·설득 순서에 집중
- 시그니처 멘트는 본인이 다듬음
- 성공 지표: 백지보다 2배 빠른 완성

**B타입 — 그대로 쓰는 완성본**
- AI 95% · 크리에이터 5%
- 말끝·호흡·시그니처까지 복제
- 성공 지표: 편집 거의 없이 업로드

### 기본값
**A타입**. B타입은 데이터 축적 후 단계적으로 접근.

**이유**: B는 크리에이터 과거 영상 STT 데이터가 최소 10편 이상 쌓여야 실현 가능. 초기에는 A로 시작해 편집 diff 수집 → 장기 B로 발전.

## 4. 해체 스키마 — 대본 분해 체계

### 4-1. 훅 (Hook) — 6유형 분류

```yaml
hook_types:
  curiosity:      # 호기심형
    pattern: "[전문 자격] + [반대 주장] + [결과 약속]"
    example: "10년 차 파티쉐가 추천하는 최고로 만들기 쉬운 디저트"
    when: "전문성·신뢰 필요한 제품"

  storytelling:   # 스토리텔링형
    pattern: "[개인 정체성] + [의외성]"
    example: "저는 무명 배우니까요"
    when: "개인 브랜드 강한 크리에이터"

  challenge:      # 챌린지형
    pattern: "[목표/상황 질문] + [과정 예고]"
    example: "혹시 지금 영업 종료했나요?"
    when: "시리즈·다회차 콘텐츠"

  negative:       # 부정/손실형
    pattern: "[손해 시나리오] + [공감 훅]"
    example: "살 잘못 빼면 얼굴 10년 늙어보이잖아"
    when: "다이어트·뷰티·건강식품 (손실회피 작동)"

  comparison:     # 비교형
    pattern: "[기존 방식] vs [새 방식]"
    example: "이 메뉴 판매량을 늘리고 싶나요?"
    when: "기능성 제품·대체재 있는 카테고리"

  visual_meme:    # 비주얼/밈형
    pattern: "[강한 시각 훅] + [짧은 설명]"
    example: "불을 제대로 끄지 않고 일단 퇴근부터 하는 모습"
    when: "숏폼 · 직관적 제품"
```

**설계 규칙**:
- 숏폼은 **7단어 이하**, 3초 안에 끝내야 함 (TikTok Shop 검증)
- Hook Rate (3초 넘긴 비율) = **핵심 KPI**

### 4-2. 본론 구조 — 카피라이팅 7대 프레임워크

| 프레임 | 약자 | 구조 | 제품 매칭 |
|---|---|---|---|
| AIDA | Attention·Interest·Desire·Action | 표준 4단 | 범용 |
| **PAS** | Problem·Agitate·Solution | 문제→확대→해결 | 기능성 (청결·다이어트·건강) |
| **BAB** | Before·After·Bridge | Before/After + 제품 다리 | 뷰티·다이어트 (시각 변화) |
| FAB | Feature·Advantage·Benefit | 기능→장점→고객 이익 | 가전·리빙 |
| 4P's | Picture·Promise·Proof·Push | 이상→약속→증거→행동 | 고관여 (가격 부담) |
| **PASTOR** | Problem·Amplify·Solution·Transformation·Offer·Response | 6단, 롱폼용 | 건강식품·교육 |
| QUEST | Qualify·Understand·Educate·Stimulate·Transition | 타겟 명확형 | 육아·니치 |

**기본 선택**: **PAS · BAB · FAB** 3종을 주력으로. 롱폼에서 **PASTOR** 차용.

## 5. 크리에이터 스타일 벡터

### 학습 차원

```yaml
creator_style_vector:
  linguistic:
    sentence_ending:     # 말끝 습관
      - "~거든요"
      - "~더라고요"
      - "~잖아요"
    signature_phrases:   # 시그니처 멘트
      - 예: "올리브영 후니언 픽입니다"
    vocabulary_level:    # 어휘 수준 (friendly/pro/hybrid)

  rhythmic:
    avg_sentence_length: # 평균 문장 길이 (음절)
    pace:                # 빠름/중간/느림
    pause_pattern:       # 쉼 패턴

  structural:
    preferred_hooks:     # 선호 훅 유형 (상위 2개)
    preferred_frame:     # 선호 프레임 (PAS/BAB/FAB 등)
    avg_duration:        # 평균 분량 (초·단어 수)

  emotional:
    tone:                # 친근/전문/유머/공감
    energy:              # 차분/보통/하이텐션
    humor_frequency:     # 유머 빈도
```

### 수집 방법
1. **온보딩 시**: 본인 영상 URL 3~5개 입력 → Whisper STT → GPT로 벡터 추출
2. **운영 중**: 생성→편집 diff 수집 → 월 단위 벡터 업데이트

## 6. Few-shot 주입 전략

### 프롬프트 구조

```
[시스템 프롬프트 — 캐시]
  · 공구 대본 기본 원칙 (G-FRAME)
  · 해당 카테고리 레퍼런스 패턴 3개 (훅·본론·CTA 각 1)
  · 설득 원칙 가이드

[유저 프롬프트]
  · 상품 정보 (카페24 API)
  · 크리에이터 스타일 벡터
  · 체험 포인트·공구 조건 (크리에이터 수동 입력)
  · 포맷 (숏폼/롱폼/시리즈)
  · 선택한 프레임 (PAS/BAB/FAB)
```

### Few-shot 예시 선정 기준 (우선순위)
1. **카테고리 일치** (뷰티→뷰티)
2. **포맷 일치** (숏폼→숏폼)
3. **톤 근접** (친근→친근)
4. **분량 유사**

### 프롬프트 캐싱
- 시스템 프롬프트 = `cache_control` 활용 (70% 비용 절감)
- 카테고리별 레퍼런스 묶음도 캐싱

## 7. 포맷 정의

### 7-1. 숏폼 (15~60초)

```
구조: Hook (1-3초) + Core (10-30초) + CTA (2-5초)

Hook      : 7단어 이하, 6유형 중 택1
Core      : SOLUTION + DEAL 통합 (장점 1-2개 + 공구 조건)
CTA       : 시한성 필수 ("오늘 자정 마감")

분량      : 40~150자 (말로 풀었을 때)
작성 규칙 : AI 보이스 지양, 크리에이터 말투 그대로 녹화 가능하도록
```

### 7-2. 롱폼 (3~8분)

```
구조: 풀 G-FRAME (Hook → Empathy → Problem → Solution → Deal → CTA)

Hook      : 20초 내 핵심 질문/주장
Empathy   : 1~2분, 크리에이터 경험담
Problem   : 기존 제품 한계, 내 고민
Solution  : 장점 3개 + Before/After
Deal      : 공구 조건 (가격·할인율·기간·수량·증정)
Trust     : 실사용 후기, 권위 인용
CTA       : 댓글/링크 안내 + 마감 강조

분량      : 800~1,500자
```

### 7-3. 게시글 (인스타 피드 · 블로그)

```
구조:
  Opening (1~2줄) → Body (여러 문단) → CTA → Hashtags

Opening   : 시선 잡는 1~2줄 (호기심형 or 공감형 훅)
Body      : 400~700자. 제품 소개 + 실사용 후기 + 장점 2~3개 + 공구 조건
            - 짧은 문단으로 나누기 (3~5줄씩)
            - 이모지 1~3개 활용 가능
            - 구체 숫자·가격 반드시 포함
CTA       : 40~100자. 구매 유도 + 링크 안내
Hashtags  : 8~15개. 브랜드·카테고리·타겟·트렌드 혼합

스타일    : 친근한 존댓말, 저장·공유 유도 멘트 포함
            ("저장해두세요", "친구한테 공유해요")
```

## 8. 품질 KPI

### 1차 지표 (스크립트 자체)
- **Hook Rate**: 첫 3초 이탈률 (목표 < 60%)
- **시청 완료율**: 롱폼 40% / 숏폼 70% 이상
- **편집 비율**: AI 생성 → 업로드 본의 diff 크기 (목표 < 30%)

### 2차 지표 (전환)
- **유효 사용률**: 쿠폰 코드 노출 대비 실사용 전환
- **완판 시간**: 공구 오픈~매진 시간
- **DM/댓글 구매 의사 표현 수**

### 3차 지표 (제품 자체)
- **크리에이터 재사용률**: 한 번 써본 크리에이터가 다음에도 쓰는 비율
- **섹션 재생성 빈도**: 어느 섹션이 가장 자주 재생성되는가 (약점 진단)

## 9. 윤리 가드레일

### 자동 검증 (생성 시 차단)
- ❌ "내돈내산" 표현 (유료 광고 인 경우)
- ❌ "100% 안전" 등 과장 주장
- ❌ 경쟁사 비방
- ❌ 의료·의학 효능 단정 ("치료된다" X)

### 자동 삽입 (선택)
- ✅ `[유료광고 포함]` · `[브랜드 협찬]` 고지 문구
- ✅ 체험 기간 명시 ("3개월 사용")
- ✅ 개인차 면책 ("효과는 개인마다 다를 수 있습니다")

### 참고 실패 사례
- 임블리 호박즙 곰팡이 → CS 부실 · 신뢰 붕괴
- 노제 광고 지연 후 삭제 → 계약 불이행
- 뒷광고 논란 → 유료 광고 미표시 법적 리스크

→ **AI는 위 사례를 재발 방지하는 문구를 자동 제안**해야 함.

## 10. TubePing 자체 프레임워크 — **G-FRAME**

7대 카피 공식·심리 10원칙·훅 6유형·홈쇼핑 5멘트·치알디니 6원칙을 **한국 공구 맥락으로 합성**한 자체 표준.

### 롱폼 G-FRAME (6단계)

```
┌─────────────────────────────────────────────────┐
│ 1. HOOK      [6유형 택1] (1-3초 / 숏폼은 7단어)   │
├─────────────────────────────────────────────────┤
│ 2. EMPATHY   [유사성+칵테일파티]                  │
│              "나도 이런 고민 있었어 / ○○분에게"   │
├─────────────────────────────────────────────────┤
│ 3. PROBLEM   [PAS.P / BAB.B]                     │
│              현 상황 구체 묘사                    │
├─────────────────────────────────────────────────┤
│ 4. SOLUTION  [FAB / 4P.Proof]                    │
│              제품 역할 + 내 경험담 + 장점 3개     │
├─────────────────────────────────────────────────┤
│ 5. DEAL      [앵커링+희소성+구체 숫자]            │
│              정가→공구가, 기간, 수량, 증정        │
├─────────────────────────────────────────────────┤
│ 6. CTA       [홈쇼핑 5패턴 택1]                   │
│              긴급/한정/인기/압박/증정             │
└─────────────────────────────────────────────────┘
```

### 숏폼 G-FRAME (3단계 압축)

```
┌─────────────────────────┐
│ 1. HOOK    (1-3초, 7단어 이하)       │
│ 2. CORE    (10-30초, SOLUTION+DEAL)  │
│ 3. CTA     (2-5초, 시한성 필수)       │
└─────────────────────────┘
```

### 게시글 G-FRAME

```
Opening → Body (HOOK + EMPATHY + SOLUTION + DEAL 통합)
        → CTA → Hashtags

인스타 피드 · 블로그용. 말로 읽지 않고 눈으로 스캔한다는 점 반영.
```

### 카테고리별 추천 조합

| 제품군 | 주력 프레임 | 핵심 심리 | 추천 훅 |
|---|---|---|---|
| 뷰티 기능성 | PAS + BAB | 손실회피+유사성 | 부정형·비교형 |
| 건강식품 | PASTOR | 권위+사회적증거 | 호기심형 |
| 가전·리빙 | FAB + 4P's | 앵커링+사회적증거 | 비교형 |
| 식품 | 스토리텔링 | 유사성+호혜 | 스토리텔링형 |
| 육아 | QUEST | 유사성+권위 | 공감형 |
| 패션 | BAB | 메라비언(시각) | 비주얼/밈형 |
